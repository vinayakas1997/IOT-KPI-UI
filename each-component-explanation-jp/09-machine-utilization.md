# 設備稼働率（稼働 / 待機 / ブロック）

## それは何ですか？
機械ごとの積み上げ棒グラフで、稼働時間を「稼働中」「待機」「ブロック」の3状態に分割します。

稼働中 — 機械が実際にサイクルを実行している状態です: 原材料を消費し、製品を生産しています。

待機 — 機械は稼働可能な状態にあるものの、まだ処理する対象がありません。前工程（またはバッファ）からの投入を待っているため、故障しているわけではないのにアイドル状態になります。

ブロック — 機械はすでに製品を完成させていますが、それを送り出すことができません。後工程（またはバッファ）が満杯、または利用できない状態だからです。準備も能力もあるのに、出来上がった製品の置き場所がない状態です。

## それを持つことのKPI価値は何ですか？
機械が稼働していなかった理由を説明し、3つの異なる問題を切り分けます。稼働率が低い場合は機械自体（サイクルが遅い、品質停止、故障）を指します。待機は上流を指します — この機械より前の工程が遅すぎる、停止している、またはバッファが不足しています。ブロックは下流を指します — この機械より後の工程が遅すぎる、停止している、またはバッファが満杯です。

大半がブロック状態の機械は通常、健全で速い機械です — それを後工程によって絞られているのです。大半が待機状態の機械は、前工程を待っている状態です。ボトルネック機械カードと合わせて読み解いてください。真のボトルネックとは、他のすべての機械がそれによってブロックまたは待機させられている機械であり、必ずしも稼働率が最悪の機械とは限りません。

## サイクルタイムへの影響
待機時間とブロック時間は、機械そのものの正味サイクルタイム（実際に稼働しているときに1個を作るのにかかる物理的な時間）を変えません。変わるのは、外部から見た実効サイクルタイムです。つまり、待ち時間も含めた1個あたりの経過時間です。

実効サイクルタイム = 正味サイクルタイム + 待機時間 + ブロック時間（1個あたり）。機械はカタログ上は優れた定格サイクルタイムを持っていても、シフトの一部が故障ではなく待機に費やされているだけで、その速度をはるかに下回る生産量になることがあります。

これはライン全体にも連鎖します。ある機械のブロック時間は、必ず1つ下流の工程の問題（遅い機械、停止、満杯のバッファ）によって引き起こされます。ある機械の待機時間は、必ず1つ上流の工程の問題によって引き起こされます。ライン上のどこか1か所で機械が遅くなったり停止したりすると、それより前のすべての工程にブロック時間として、それより後のすべての工程に待機時間として現れ、工程間のバッファが小さいと、その波及効果は数分でライン全体に及ぶこともあります。

## それを計算するために何が必要ですか？
単純な稼働／停止フラグではなく、PLCまたはMESからの機械ごとの連続的な状態信号（稼働中／待機／ブロック）。3つの状態はどの瞬間においても互いに排他的であり、合計するとシフト時間のほぼ100%になるはずです。

ある状態の割合 = その状態で過ごした時間 ÷ シフト総時間。

稼働率をサイクルタイムへの影響に結び付けるためには、機械ごとの正味サイクルタイムも取得し、「ブロックまたは待機のせいで遅い」のか「工程自体が遅い」のかを切り分けられるようにします。
