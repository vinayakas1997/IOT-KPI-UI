# MTBF・MTTR — 信頼性指標

## それは何ですか？
シフトタイムライン上の各機械の行の隣に表示される2つのサマリー数値、MTBFとMTTR、そしてカードの上に表示される小さな「開始: `<date>`, `<time>`」ラベルです。このラベルは、これらの数値が計算されている期間の起点を示します — 現時点では、その期間は1シフトです。

MTBF（平均故障間隔）は、機械が再び故障するまでに稼働し続けられる平均時間です。高いほど信頼性が高いことを示します。

MTTR（平均修復時間）は、故障してから機械が復旧するまでにかかる平均時間です。低いほど修復対応が速いことを示します。

ここで故障としてカウントされるのは計画外の停止のみです。休憩、段取り替え、計画保全といった計画された停止は、タイムライン上で別途バッファ時間として扱われ、どちらの数値にもカウントされません。

## それを持つことのKPI価値は何ですか？
MTBFとMTTRは異なる事象を測定しており、それぞれ異なる対策を示します。MTBFが低い、または縮小している場合は機械の故障頻度が増えていることを意味し、修復の迅速化ではなく、予防保全と根本原因の排除に投資すべきです。MTTRが高い場合は、故障がどれほどまれであっても、発生するたびに多くの時間を消費していることを意味し、信頼性工学ではなく、予備部品の確保、技術者の対応時間、診断能力の向上に投資すべきです。

機械によっては、一方が強く他方が弱いということもあります。めったに故障しないが直すのに何時間もかかる機械もあれば、頻繁に故障するが数分で復旧する機械もあります。この2つの数値をタイムラインのブロックパターンと並べて読むことで、単一の統合された信頼性スコアに頼ることなく、どのような故障プロファイルなのかを把握できます。

「開始: `<date>`」というラベルが存在するのは、これらの数値が実際以上に権威あるものとして読まれないようにするためです。MTBFとMTTRは平均値であるため、多くのシフトにわたる多数の故障をもとに計算されて初めて統計的に意味を持ちます — 1シフトに数件程度の故障は、安定した信頼性の数値として信頼するにはサンプル数が少なすぎます。故障が0件または1件のシフトでは、MTBF／MTTRがシフトごとに大きく変動します。これは1シフト分のサンプルとして予想される変動であり、数値が誤っているわけではありません。個々のシフトの絶対値ではなく、多くのシフトにわたる傾向を信頼してください。

## それを計算するために何が必要ですか？
共有のダウンタイム／アラームイベントリストの停止エントリを機械ごとに、シフト時間内に限定して導出します。

故障件数は、そのシフトにおけるその機械の計画外停止ブロックの件数です。総停止時間はその継続時間の合計です。

MTBF = (シフト長 − バッファ時間 − 停止時間) / 故障件数。バッファ時間と停止時間はまずシフト長から差し引かれるため、計画された停止は故障としてカウントされることも、MTBFの基準となる「稼働」時間に含まれることもありません。

MTTR = 停止時間 / 故障件数。

シフト中にまだ故障が0件の場合、割る数がないため、MTBFは稼働時間そのものにフォールバックし、MTTRは0と表示されます — これは「まだ故障なし」の状態であり、文字通りの修復時間ゼロを意味するものではありません。

「開始」ラベル自体には固有の計算はありません — この演算がすでに使用しているシフト日付とシフト開始時刻をそのまま反映しているだけです。これを実際の複数シフト・複数日にまたがる追跡期間に拡張するには、複数日にわたる故障・ダウンタイムイベントを蓄積する仕組みが必要ですが、現在のデータモデルにはまだそれが備わっていません。
